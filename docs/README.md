# Repository Philosophy

This repository focuses on **production-ready detection engineering** for CrowdStrike Falcon Next-Gen SIEM.

Unlike generic query collections, every detection in this repository follows a standardized engineering lifecycle.

```
Threat Research
        │
        ▼
Threat Modeling
        │
        ▼
MITRE ATT&CK Mapping
        │
        ▼
Telemetry Analysis
        │
        ▼
Detection Logic
        │
        ▼
Query Development
        │
        ▼
Validation
        │
        ▼
False Positive Tuning
        │
        ▼
Production Deployment
```

Every detection should be:

- Behavior-based
- MITRE ATT&CK mapped
- Performance optimized
- False-positive tuned
- SOC investigation ready
- Validation documented

# Repository Components

## Detection Rules

Detection rules are organized according to MITRE ATT&CK tactics.

Example:

```
detections/
    TA0007-Discovery/
        DISC-001-Unauthorized-Host-Recon/
```

---

## Lookup Framework

Lookups are used to reduce false positives by identifying approved assets, privileged users, service accounts, jump servers, and other trusted entities.

---

## Validation

Every detection includes a validation guide describing:

- Test commands
- Expected Falcon telemetry
- Expected detections
- Investigation workflow

---

## Playbooks

SOC investigation playbooks explain how analysts should investigate alerts generated by each detection.

---

## Threat Hunting

Threat hunting queries complement detection rules by enabling proactive investigation of suspicious behavior.

---

# Detection Engineering Principles

This repository follows these principles:

- Detect attacker behavior rather than individual tools.
- Minimize false positives.
- Maximize detection coverage.
- Prefer correlation over single-event detections.
- Use lookup tables wherever practical.
- Validate every rule before production deployment.

---

# Rule Quality Requirements

Every production rule should include:

- Detection query
- MITRE ATT&CK mapping
- Threat scenario
- Required data sources
- Required lookup files
- Validation guide
- Investigation guide
- False-positive analysis
- References
- Version history

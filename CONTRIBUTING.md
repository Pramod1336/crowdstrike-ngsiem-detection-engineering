# Contributing

Thank you for your interest in contributing to the CrowdStrike NG-SIEM Detection Engineering project.

## Ways to Contribute

- Detection Rules
- Lookup Templates
- Validation Guides
- Threat Hunting Queries
- Investigation Playbooks
- Documentation Improvements
- Bug Fixes

---

## Detection Rule Requirements

Each new rule must include:

- Rule Name
- Description
- MITRE ATT&CK Mapping
- Data Sources
- Detection Query
- Validation Steps
- False Positive Analysis
- Investigation Guide
- References

---

## Pull Request Checklist

- [ ] Query validated
- [ ] MITRE mapping added
- [ ] Documentation complete
- [ ] Validation steps included
- [ ] False positives documented
- [ ] References included

---

## Coding Style

- Use descriptive names.
- Keep queries readable.
- Avoid environment-specific values.
- Use lookup files whenever possible.
